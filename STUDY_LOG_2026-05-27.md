## Drill 3 — Rolling Average

```python
smooth = df['dT_S07'].rolling(window=30, center=True).mean()
# center=True: window is centred on each point (no lag)
# center=False (default): window looks backward (introduces lag)
```

**What it does:** smooths noise while keeping the original time index.
`window=30` with 1-min data = 30-minute smoothing.

**rolling().std():** tells you how variable a signal is over time.

**Practice:** Compute 60-min rolling mean for ALL `sub_cols` at once:

## Drill 7 — GridSpec: Custom Multi-Panel Layouts

```python
import matplotlib.gridspec as gridspec

fig = plt.figure(figsize=(15, 8))

                        width_ratios=[2, 1],   # left panel 2x wider
```

## Problem 1 — Two Sum (LeetCode #1)

**Problem:** Given `nums` and `target`, return indices of two numbers that sum to `target`.

**Brute force O(n²):** two nested loops — check every pair. Too slow.

**Optimal O(n) — Hash map:**

```python
def two_sum(nums, target):
    seen = {}                         # {value: index}
    for i, num in enumerate(nums):
        complement = target - num
        if complement in seen:
            return [seen[complement], i]
        seen[num] = i
```

**How it works:** for each number, check if its complement (`target - num`) is already
in `seen`. If yes → found the pair. If no → store current number.
One pass, O(1) lookup per step.

**Pattern to remember:** `complement lookup in a dict` — appears in many array problems.
If you see "find two elements with property X", think hash map first.

**Practice from memory:** write it without looking. Test with `[2,7,11,15], 9` → `[0,1]`.

## Problem 2 — Valid Anagram (LeetCode #242)

**Problem:** Return `True` if string `t` is an anagram of string `s`.

**Sorting O(n log n):**
```python
return sorted(s) == sorted(t)
```
Clean but slow.

**Optimal O(n) — Counter:**
```python
from collections import Counter
def is_anagram(s, t):
    return Counter(s) == Counter(t)
```

**How it works:** `Counter(s)` builds `{char: frequency}` dict in O(n).
Two strings are anagrams iff every character has the same frequency in both.

**`Counter` tricks to know:**
```python
Counter("hello")              # Counter({'l': 2, 'h': 1, 'e': 1, 'o': 1})
Counter("hello") - Counter("hell")  # Counter({'o': 1})
Counter("a") + Counter("b")   # Counter({'a': 1, 'b': 1})
Counter("hello").most_common(2)     # [('l', 2), ('h', 1)]
```

**Works on any iterable:** lists, tuples, Thai strings, numpy arrays.

**Practice from memory:** write the one-liner. Then try without Counter using a dict manually.

## Problem 3 — Best Time to Buy and Sell Stock (LeetCode #121)

**Problem:** Given price array, find max profit from one buy + one sell (buy before sell).

**Optimal O(n) — Running minimum:**
```python
def max_profit(prices):
    min_price  = float('inf')
    max_profit = 0
    for price in prices:
        min_price  = min(min_price, price)
        max_profit = max(max_profit, price - min_price)
    return max_profit
```

**How it works:** at each day, the best profit *if you sell today* = `price - min_so_far`.
Track the running minimum and the running max profit simultaneously.

**Generalised pattern — "running minimum scan":**
```python
running_min = float('inf')
for val in array:
    running_min = min(running_min, val)
    # best_result_at_this_point = f(val, running_min)
```
Use this when: best entry point, first threshold, max gain from a baseline.

**Practice from memory:** write it. Test with `[7,1,5,3,6,4]` → `5` and `[7,6,4,3,1]` → `0`.

## SQL Problem — Duplicate Job Listings (DataLemur Easy)

**Problem:** Find the number of companies with duplicate job listings
(same company_id + title + description appearing more than once).

**Solution:**
```sql
SELECT COUNT(DISTINCT company_id) AS duplicate_companies
FROM (
    SELECT   company_id
    FROM     job_listings
    GROUP BY company_id, title, description
    HAVING   COUNT(*) > 1
) AS dupes;
```

**Step by step:**
1. `GROUP BY company_id, title, description` — groups identical postings together
2. `HAVING COUNT(*) > 1` — keeps only groups that have duplicates
3. `COUNT(DISTINCT company_id)` — counts unique companies (one company could have multiple duplicate pairs)

**SQL patterns to memorise:**

| Pattern | Syntax | When to use |
|---------|--------|-------------|
| Filter after aggregation | `HAVING COUNT(*) > 1` | Duplicates, groups meeting a threshold |
| Filter before aggregation | `WHERE col = 'x'` | Row-level filtering |
| Subquery as table | `FROM (...) AS alias` | Chain two aggregations |
| Count unique | `COUNT(DISTINCT col)` | Unique companies, users, products |
| Detect duplicates | `GROUP BY + HAVING COUNT(*) > 1` | The universal duplicate-detection pattern |

**Practice:** re-write the SQL from memory. Then try: "Find companies with at least 3 job listings total."
```sql
SELECT company_id, COUNT(*) AS total
FROM   job_listings
GROUP BY company_id
HAVING COUNT(*) >= 3;
```
